搭建博客

###1，首先下载git:
https://git-scm.com/download/win
默认安装即可。
安装后在桌面右键会有 Git Bash Here

###2，安装hexo 这是官方中文文档https://hexo.io/zh- cn/docs/
在桌面右键 Git Bash Here 打开命令模式：
npm install –g hexo

###3，在任意盘创建文件夹（你喜欢的都可以：C/D/E/F/G）命名为hexo（其实什么都可以的）
进入到你所建文件的目录，记住是你刚创建的命名是什么我就不知道了，我的是hexo，鼠标右键Git Bash 也会出现命令模式

###4，生成文件：
hexo init
安装依赖包：
nmp install

###5,上述命令成功完成后，可以看见hexo目录下新建了好多的文件，文件夹的。
启动服务：
hexo generate(其实 hexo g 效果是一样一样的)
hexo server
OK，本地静态博客已经搞定了，不信你在浏览器输入localhost:4000看看效果，是不是吓着你了，哈哈！

###6，然后就是把博客部署到GitHub上，（至于为什么要选择GitHub，毕竟人家牛逼，关键又不要钱，也不限流量，问什么不呢？）

###7，关于注册GitHub上创建账号就不多说了，作为一计算机专业，不会这，感觉好丢人的。

###8,创建repository（在右下角）

###9,创建一个repository，这里的话就是重点了，一定要是你GitHub名格式：
你的名.github.io 明白？

###10，将本地的静态部署到GitHub上，进入你刚在什么盘创建的什么目录（我真的不知道），有一个_config.yml的可编译文件，想怎么打开就怎么打开，打开就可以，修改下面几个地方：
deploy:
type: git
repository: git@github.com:xiaofengtongxue/xiaofengtongxue.github.io.git
branch: master
值得注意的是，每一个： 都有一个空格的。

###11，然后就是完成部署：（依旧是刚才你创建的什么文件夹下）
hexo g
hexo d
OK,基本搞定了，第一次需要点时间，然后在浏览器访问你的：你的名字/你的名字.github.io
看看有没有效果，是不是傻了你。

###12，后续的优化再说吧，域名的绑定也很简单的，现在买个域名挺便宜的，十块钱左右，写个文件，做个DNS解析就可以的。

##OK，巴拉完了。
