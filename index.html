搭建博客

1，首先下载git:

https://git-scm.com/download/win
默认安装即可。
安装后在桌面右键会有 Git Bash Here

2，安装hexo 这是官方中文文档https://hexo.io/zh- cn/docs/

在桌面右键 Git Bash Here 打开命令模式：
npm install –g hexo

3，在任意盘创建文件夹（你喜欢的都可以：C/D/E/F/G）命名为hexo（其实什么都可以的）

进入到你所建文件的目录，记住是你刚创建的命名是什么我就不知道了，我的是hexo，鼠标右键Git Bash 也会出现命令模式

4，生成文件：

hexo init
安装依赖包：
nmp install

5,上述命令成功完成后，可以看见hexo目录下新建了好多的文件，文件夹的。

启动服务：
hexo generate(其实 hexo g 效果是一样一样的)
hexo server 
OK，本地静态博客已经搞定了，不信你在浏览器输入localhost:4000看看效果，是不是吓着你了，哈哈！

6，然后就是把博客部署到GitHub上，（至于为什么要选择GitHub，毕竟人家牛逼，关键又不要钱，也不限流量，问什么不呢？）

7，关于注册GitHub上创建账号就不多说了，作为一计算机专业，不会这，感觉好丢人的。

8,创建repository（在右下角）

9,创建一个repository，这里的话就是重点了，一定要是你GitHub名格式：

你的名.github.io  明白？

10，将本地的静态部署到GitHub上，进入你刚在什么盘创建的什么目录（我真的不知道），有一个_config.yml的可编译文件，想怎么打开就怎么打开，打开就可以，修改下面几个地方：

deploy:
 type: git
 repository: git@github.com:xiaofengtongxue/xiaofengtongxue.github.io.git
 branch: master

值得注意的是，每一个： 都有一个空格的。

11，然后就是完成部署：（依旧是刚才你创建的什么文件夹下）

hexo g
hexo d
OK,基本搞定了，第一次需要点时间，然后在浏览器访问你的：你的名字/你的名字.github.io

看看有没有效果，是不是傻了你。

12，后续的优化再说吧，域名的绑定也很简单的，现在买个域名挺便宜的，十块钱左右，写个文件，做个DNS解析就可以的。

OK，巴拉完了。

黑客与画家

忘记什么时候逛学校图书馆时，借的《黑客与画家》这本书，直到前几天才看完，这是一本能引发技术人思考的不错的书，黑客、创业、编程语言，是这本技术散文集的三个主题。当然我借的是一本翻译版，感觉翻译国外许多经典书籍是一件很不容易的是，毕竟不是作者，很难表达作者本来想表达的情感，不过这本书却翻译的不错，看的很投入。看完后就想着做两件事：1，非常想创业（虽然我知道我不行，但是男人怎么可以说自己不行呢？），2，就是立志当一名黑客，不知道为什么，也许是说真正的黑客无所不能吧。

这本书总体解释为什么书呆子不受欢迎到阐述黑客精神的意义，实际上是给黑客进行了正名。保持黑客精神，就有可能改变这个世界。那些不服从管教的人们，是这个社会力量与财富的源泉，说的不错，当今社会很少有人关心这些了，大家更关心房子，车子和报酬。

我是一个即将毕业的大学生，“如何创造财富”这张对我的影响很大，技术人有必要建立起对财富的价值观。说到钱，估计是很多人不愿意提的话题，或许是碍于面子吧。但是又是看到很多技术牛人都在谈论股票，谁让我们生活在这个糟糕的时代呢？很少有人能说清楚财富和技术的关系，还好有这本书。金钱只是财富的一种表达方式，但财富不等于金钱，不止是房子与车子。

保罗•格雷厄姆的观点是：致富的最好办法是创造财富（而不是掠夺），自己创业或是加入创业团队是致富的可靠方法（在我看来，你家里如果是特别有钱的话，那就别说话了！）。给我带来不小反思的还有关于“不能说的话”的那章节。身处当前这种看脸的社会，如果你发现了这个社会的某种禁忌，发现了不能说的话，怎么办？最恰当的办法就是在合适的地点合适时候说（感觉是我的话，什么都不会考虑的，还就不信呢有什么不能说呢，我就不明白了！），而不是到处去说，我们要学会“只打值得打的仗”。想想我们平时在玩游戏的时候耗费大量精力发动的口水战，难道不是无谓之争么？作者还提示了一个办法，那就是幽默（作者真的太NB，不知道是不是翻译的原因？）。如果自己是潮水的一部分，你无法看清潮水的方向，唯一的办法是永远保持质疑，提升自己的思辨能力。

后来百度了一下作者，发现作者以前是一位画家（可能是那种业余画家，其实我也可以算的上是一个业余的业余画家，哈哈，我是来搞笑的！），同时还是一位天使投资人，创业导师等，越来越崇拜他了，也是像他这种从成功走向成功的人很少的，好多人都是失败，失败，然后在失败中找到窍门，从此走向成功。不错精通一门技术确实很难，要学好两门以上的技术，感觉不仅需要强大的毅力，更需要一个长久而又完美的计划。写程序的人都是天才型的，如果有一天你发现天才不是唯一值得高兴的是，那就更好了。或许每个人想法不一样，这只是我个人的一些观点（不喜欢的话，有本事来打我啊!）。

作为一个十足的本理科生，并没有什么好的文笔，而却从高考到现在，3年几乎没有没怎么动过笔，已经不是从前那个少年了，如今的我，作为一名大学生的我，每天除了上上课，玩玩游戏，打打台球一类的，没什么别的爱好。大部分时间就是待在寝室，偶尔做做与专业相关的事情，其实很不想和别人谈论自己的专业，因为我上的是一个不伦不类的专业，至少我觉得是，之前都是这样认为的。有时间我还会重新读这本书中章节，找到不一样的感受，反思。

巴拉巴拉完了，大概是中学语文没学好吧，也就是这个样子了。哈哈！！！

我的青春有你

昨天是科比最后一场比赛，由于昨天没时间的原因，没有及时写写自己的小感悟

首先说说昨天的比赛，无非是一场史诗级的告别战，38岁的科比在NBA生涯最后一场比赛中毫无保留，出手50次创新高，狂劈60分成为NBA历史单场60+年纪最大的球员，打破NBA球员退役战得分纪录。科比NBA生涯第六次60+，在NBA历史60+场次榜上位列第二，仅次于张伯伦。科比仿佛穿越时空，回到无坚不摧的巅峰期，一代传奇以这样的方式转身离开，没有比这更圆满了。

再回顾一下他拥有的荣誉，NBA7大荣誉全包揽：在科比20年的职业生涯中，与乔丹一样，实现了总冠军、常规赛MVP、总决赛MVP、全明星赛MVP、得分王、最佳阵容和最佳防守阵容7大荣誉大满贯。这其中包括一次常规赛MVP，两次总决赛MVP，4次全明星MVP与佩蒂特并列历史第一。

此外，他在成为湖人五冠王的同时，还拿到了包括得分王在内的8项队史第一，可以这样讲，他就是紫金军历史头号球星。对于联盟，科比NBA生涯15次入选最佳阵容，历史第一；12次进入最佳防守阵容，NBA历史中仅次于邓肯（15次）。科比还2次成为赛季得分王，他在2005-06赛季场均35分领跑全联盟，这是1987年至今NBA球员单赛季场均最高分。

自己以前上中学的时候也是球迷，自己的青春伴随着科比长大，感谢在我人生最需要精神支柱的阶段——科比，你的出现。让我明白的人生是不可以说放弃的，需要怎样的去生活。以前遇到点什么事，首先想到的是怎么逃避，而你，在一次次的痛苦中，表现出的不是退缩，而是静静的去享受这种痛苦，也享受伤口重新长齐后的粗壮伤疤和神经。活在这样的世界里，也去就是你的生活方式，也是我喜欢的原因所在。你是我们年轻的精神寄托，我们让你代替我们在糟糕的现实世界里保持倔强，拒绝改变，冷对嘲讽，然后疯狂打脸。想到这感觉自己与科比最大的差距，就是在比赛仅剩3分20秒还落后10分时，我认为可以放弃了而他没有。

记得科比在2015年年11月底宣布即将退役的，整个中国球迷、整个NBA就开始了与科比漫长的告别。虽然科比集爱恨于一身，但最终他的“告别巡回”还是温馨的。当年那些恨他入骨的球迷很多都放下了恩怨，在他一次次离场的时候报以热烈的欢呼和呼喊。昨天，这一切终于走向了漫长的终点。我们用已经成熟世故的眼光再次看着这个偏执的疯子，就像看待当时还愣头愣脑的自己。

Linux重要文件

Linux系统系统目录

/etc/passwd:

该文件信息是由6个分号组成的7个信息:（从左到右）
1:用户名
2:密码(已经加密)
3:UID(用户标识),操作系统自己用的
4:GID组标识
5:用户全名或本地帐号
6:开始目录
7:登录使用的Shell,就是对登录命令进行解析的工具。

/etc/shadow:

该文件信息是由8个分号组成的9个信息:（从左到右）
1，账号名称
2，密码（已加密）
3，最近更改密码的时间
4，密码不可被更改的天数
5，密码需要重新变更的天数
6，密码需要变更期限前的警告期限
7，密码过期的怒限时间
8，账号失效日期
9，保留

/etc/group:

该文件信息是由3个分号组成的4个信息:（从左到右）
1，用户组名称
2，用户组密码
3，GID
4，用户列表

Linux基础操作

自己学习Linux基础：

我给自己虚拟机上安装CentOs（玩软件的，都玩过虚拟机吧！），首先是配置IP，DNS，通过Xshell连接虚拟机(开启ssh服务)，创建用户设置权限等。

下面是几条常用到的命令：

00x1：手动临时配置

Ifconfig eth0 up/down ：开启/关闭 eth0网卡
Ifconfig eth0 192.168.1.110 （netmask 255.255.255.0）：设置eth0网卡的IP地址（此时设置的IP是临时的，重启消失）
Ifconfig：查看与设置IP地址，子网掩码
route add/del default gw 192.168.1.2 ：设置默认网关为192.168.1.2

00x2：文件写入永久配置

/etc/sysconfig/network-scripts/ifcfg-eth0/eth1 （此文件中添加ONBOOT=yes,IPADDR,NETMASK,GATEWAY等）
/etc/resolv.conf ：设置DNS服务器
service network start|restart|stop ：重启网卡

00x3：安装一些基础软件包

安装MySQL命令：yum –y install mysql-server
开机自启：chkonfig mysql on
安装Apache命令：yum install httpd
开机自启：chkconfig –levevs 235 httpd on
安装Php命令：yum –y inastall php
目录：/var/wwww/
phpMyadmin目录：/usr/share/phpMyAdmin
Apache配置：/etc/httpd/conf.d/php.conf

虚拟机挂载硬盘：

感觉这个不是非常重要，但是要会的…

00x1：手动临时挂硬盘

首先在虚拟机虚拟网卡上添加一块硬盘，然后重启虚拟机，运行fdisk –l 查看虚拟机主机硬盘的分区情况（如果之前是虚拟机安装的是一块硬盘，就会发现会多出一条信息，Disk: /dev/sdb 存在未分区的情况），然后就需要进行分区操作：
fdisk /dev/sdb 机会让你选择对该硬盘进行操作 m（帮助） n（新建一块分区） d（删去硬盘上分区） w（将分区写在硬盘上 前提是已经将硬盘分好区了）
再次运行上面命令发现硬盘已经分区了，df –lh 看到硬盘并没有挂在到任何目录下，这是肯定的，因为还没有对硬盘进行挂载
挂载命令：
mkfs.ext3 /dev/sdb1 是对刚分好的区进行格式化
mount –t ext3 –o rw /dev/sdb1 /home/fans 该命令是将/dev/sdb1硬盘挂载到/home/fans目录下（前提是在/home/下已经创建过fans目录，cd到该目录，命令mkdir 文件 进行创建）

00x2：永久性挂载硬盘

查看刚建分区的SSID：blkid /dev/sdb1
在/etc下vim fstab 将分区的信息添加在文件中，重启生效 

进程查看：

ps 查看系统进程
最好的参数 ps –aux （查看系统所有的进程）
top 以动态的方式进行动态的显示系统进程（跟win系统的任务管理器差不多的）
以上方式都可以显示进程的PID号
kill 终止进程
kaiiall 终止由一个软件开启的多线程（可以直接跟软件名的）
